Dimensionality reduction is quite a large topic, with many different
models covering many different facets and ways to find low dimensional
manifolds contained within the high dimensions of our data
This note is a follow up to such notes to provide even more modern examples
of such methods ^xsIk76Bx

Multi-Dimensional Scaling
(MDS) ^Y1a25r0U

Minimizes the discrepancy between the pairwise distances in the original space
and the pairwise distances in the low-D space.
If we can find the mapped low dimensional points such that the overall discrepancy
between their distances and the original points is minimal then we have correctly
mapped the points to low dimensional space while keeping the same structure ^shRHJqjI

Reconstruction ^xxkArZBb

mapped x vector
to low dimensional
space ^nqY4OYb5

The approach for this algorithm is to start with random mapped low dimensional
points and perform Gradient descent until you reach a local minima ^Q6zG17xt

If the data-points all lie on a hyperplane, their pairwise distances are
perfectly preserved by projecting the high-dimensional coordinates onto
the hyperplane ^6frDdAOH

MDS is equivalent to PCA if we "Double center" the data
But this will create spurious structure. ^iMp7lShw

Double centering means making the mean value of every row and
column be zero, i.e centering both the samples and the features ^X43MRke3

auto-encoders can be non-linear which makes
them much more powerful than PCA but much slower
and harder to optimize ^tdbYlK8K

This method can be made non-linear by quantizing distances
with unequal importance, i.e far distances get less and less importance of
accuracy whereas low distances have to be really accurate ^9o3cRfST

slow to optimize and also gets stuck in different local optima each time. ^dOyjhpgJ

IsoMap ^tUFOayun

Instead of only modeling local distances, we can try to measure the distances along the manifold
and then model these intrinsic distances.
The main problem is to find a robust way of measuring distances along the manifold. ^KQYG7qTs

2. Put the true Euclidean distance on each of these links.

3. Then approximate the manifold distance between any pair of
points as the shortest path in this “neighborhood graph” ^T478uKNf

Gradient ^DEO5zzVV

Physically the gradient may be interpreted as a resultant force created by
a set of springs between the map yi and other maps yj
all springs exert a force along the yi-yj direction, the force is either repelling
or attracting depending if the distance between them is too large
or too small to represent the original points ^u7AcwHMg

Crowding Problem ^34aCceHd

Gaussian distributions become really close 0 the further
you go to the right (large distances) which means it will
exert more and more force from points that are really far
which crushes the rest of the map up ^wOfUZhEx

We can use a heavily tailed distribution (dont tend to 0 very fast)
 instead of a gaussian in order to remove that effect ^NITxXPe6

What about Non-Euclidean Pairwise
Distances? ^hsQSlZk3

More often than not we also want to find relations and manifolds of data that is non-euclidean
The reason this data is non-euclidean is because it doesnt follow euclidean rules and cannot be represented
as vectors, but instead has to stay as metadata. ^SvmVIfgO

Given the dissimilarity matrix D such that dij is the euclidean distance between xi and xj
dij has to adhere to the following rules for it to be measuring metric data:

1) dij >=0, distance cant be negative

2) dij=0 only if i and j are the same

3) dij=dji distance between two points is the same
no matter where u look at it from

4) dij <= dik + dkj Triangle inequality ^nAoqguq7
