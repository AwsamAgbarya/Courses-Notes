Keyframe animation ^RjmUIpVJ

We define objects at distinct points in time by specifying the
transformation occuring OR the goal vertex positions called a keyframe.
because its a frame of reference that is key to our animation.
We interpolate OR tween between the keys in order! ^L10SPhit

Rigid body transformation ^K6vQg0kx

Only change the local to global transformation in the scene graph as a function of time ^3VG7cjbb

for translations we perform linear interpolation between 2 points in time, for rotation we use SLERP
 ^Aern7AEu

Tweening ^m5KDwa8D

Quite similar to rigid body transformations, such that we
want to interpolate the position between two points in time
and perform slerp with quaternions from one orientation to another
BUT this time we perform it with an ease-in ease-out mechanism
such that it looks smoother ^cHgQ9LKe

No longer linear ^T7Nib7Kp

t to a negative value or a value greater than 1 leads to undershooting to create a bouncing effect ^tbIHWrpG

Vertex Animation ^whdZh83d

Specifying the positions of vertices in a set amount of keyframes and interpolating
between these positions ^0FxzcfCw

problem:
 that this on its own causes an unwanted effect of shrinking and growing between keyframes ^fdPdCyYB

Joints and limbs then we can perform Skeletal animation in order to fix this issue ^DsXItLZj

Skeletal animation ^KridRNV8

We do not need a keyframe for every single vertex position
instead we describe our motion as a set of joints in a hierarchical manner
and describe the keyframes using those joints
(ALSO CALLED RIGGING)

bones might affect multiple vertices differently thus we
bind different weights to each vertex describing how each bone moves it ^7GmOjO8c

wij describes how much vertex i is influenced by bone j
if this weight is one then it is rigidly attached to it!
usually the amount of bones that can influence a vertex is limited ^xpB3vW9B
