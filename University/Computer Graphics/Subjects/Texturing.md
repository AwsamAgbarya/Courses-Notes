Texturing ^p2LfCZXe

What are textures? ^R7fsObci

Models in the world require a lot of detail especially on a smaller scale
those details are often way too expensive to be modeled using primitives
thus we embed textures using Texels while rendering the objects. ^DTvq4XHv

How is it done? ^iOdjNoAe

We save texture information in a datastructure
we define a function that maps 1-1 point on primitive to
a texture
Paste the info in when rendering ^2UOKFQmW

Mapping coordinates in world space xyz
onto coordinates in texture space uv
F_invmap ^vNmDMh8v

Map texture coordinates uv into rgb values
in order to be rendered and colored using
C_tex ^98TuMXZI

Can be described as a three vector Cij in a
2d matrix for each texel ^84z5DxrR

Sphere Mapping ^ezN5hua6

Using polar coordinates theta and phi we can map any point
on a 3D sphere into two angles, such that those two
angles will get mapped to u,v ^DNKyNvSl

Cylinder mapping ^pPApVPq3

We assume the cylinders axis is parallel to z axis
that way we can describe the z value using h
x and y values map to an angle from the axis
which is mapped to u,v ^saSQVghL

Box Mapping ^wQqa5V61

We surround our object with a bounding box based off of max and min
values, then we need to identify our 2 major axis and omit the third
that way our 2 major axes map to uv coordinates ^tRIFotIv

General Mapping ^9EZcHpCR

Typically done manually by the artist where we map each part of the
texel file onto a specific vertex by specifying its boundaries ^UNAo5FQi

Volume mapping ^nU0QW8W0

What about mapping things in spatial coordinates uvw such that we dont
need a mapping function from world coordinates, only a color function.
this is computationally cheaper but much more expensive on storage ^rlFEyYDi

Texels ^aNoDuPwI

We want to evaluate the pixel color when we lie in-between
different Texels during magnification and minification,
 this has many strategies some of which are better than others! ^C1QJEh11

Naive Solution for Magnification ^GgT0AZRp

Nearest Neighbour ^keefhUwt

Always round down to the integer values of the texels
you have in the border of your pixel!

 ^wv3M9rLK

u ^3dA2X8Id

v ^zB2Ch6JA

This will not produce nice results as all values within that pixel will
be one color which creates pixelated images and artefacts that are very

A different kind of interpolation! ^FIcPuCiq

Bilinear Interpolation ^Q2X194x1

u ^KNd43a1O

v ^eJN1540E

1 ^LZKMGsOu

2 ^SCEyTwu4

3 ^TbuBao9P

We interpolate a value between 1 and 4
which is a mixture "middle ground" of both colors
We interpolate a value between 2 and 3
which is a mixture "middle ground" of both colors

Then we Interpolate between the first color
and the second color, that way we get an interpolation
of all 4 colors into 1! ^VR1UPW5b

Solution for minification ^5ZYB78cM

For minification it is insufficient to compute one color
value per pixel as there will be a lot of details that will be missed within
that pixel, under sampling leads to aliasing ^JhRb9GFk

MipMapping ^480LlV1s

Create a hierarchy of textures
At level d of the texture hierarchy, texels of the original texture are
 captured by the value 2^2^d of one texel
we choose this texture level based on minification level
and its potentially interpolated between levels ^xuKRSEzi

Preform sum pooling in order to get the higher level ^GjjzNBkp

First we calculate the pixel footprint! ^m9cr4Irw

Footprint: projection of a square pixel into the texture coordinate space ^jDhMmx7D

We preform pixel corner translation from pixel space onto texture space

Using derivatives of the texture coordinates uv w.t.r screen space coordinates
We can calculate the difference in movement from one corner to another ^b6Y252PN

We preform the calculation for both a and b and pick out the maximum one
in order to decide our level of mipmapping ^ThrF63by

If d is not an integer it is rounded
or
it is interpolated  ^a6JlSgLD

How do we store mipmaps? ^MCrAXqvi

Mipmaps only really take up 1/3 more memory than
normal textures, using this method of initializing
the rgb map to 2x(size of texture) ^XAK2SqLd

Mipmapping using footprints is anisotropic
another method of performing anti aliasing is using SAT ^emRqNgfT

Summed area Tables ^cIV3dndd

SAT is a technique that uses dynamic programming techniques in order to
approximate axis-parallel rectangles
we first calculate the axis-parallel bounding box of our footprint
and we use our precomputed summed area table in order to perform the approximation ^SnZ6ubo9

How do we precompute it? ^yVgaR1cf

We go over a matrix the size of our texture
for each pixel we calculate the sum of the area of the
square from 0,0 up to it, by using previous values we calculated
and summing them up ^AWh3wxeh

Rasterization ^Zx0wfVzb

Rasterization of textures over primitives requires interpolation of uv values which are
only stored in vertices
BUT!
Rasterization is done after perspective projection!
simple interpolation leads to wrong results ^GKK0IKuJ

Are we only mapping colour for textures? ^YfByMwoP

NO! ^Zlt7Iy8d

Light mapping ^3HcgPWDK

for relatively static environments in terms of light, we can often
compute and store the light in a separate light map that matches
the light value of each texel

This is only storing the view independent part of the light!
more about it in radiosity

 ^R1mLHg2H

Environment Mapping ^Sbmb4GDC

Modelling mirroring effects in static environments can sometimes
be computed and stores in a map too
this can be done in small reflecting objects such that depending
on the angle of viewing the object it gets mapped to a different
uv coordinate which reflects the environment in the room! ^Yc4VoWfF

Matcaps ^0bJuCZZi

matcaps are material textures that capture the lighting
of the material as well, it is often modeled as a circle which depending
on the angle of the norms on the material gets mapped to a different point
on the circle reflecting a certain amount of light ^1jKcOxwt

Height Mapping ^ebiLL3hv

In rougher materials, it isnt enough to give them color texture, we are often requires
to add depth to those textures to simulate a rough rocky surface or even just
a realistic non planar surface. ^LzqIZjco

Displacement mapping ^bnkTgYDm

Introduce new geometry based on the height fields of the map.
each norm on the surface has a different height and thus we are required
to calculate the new norms given by the displacement of those norms and add
new geometry to simulate the new rough surface ^2ql0Zc26

Leads to significant overhead ^eiGZ4nUr

Parallax Mapping ^KTXJwQFW

Compute the new normals using the height field map but do not use them for rendering
instead only use them for lighting computations which causes the same shadowing
effect except the material itself is still flat ^4Jg18kq0

Normal mapping ^uTGQGgTX

Preform the same computations as Parallax offline
and store it in a heightmap that is then combined in the rendering
stage ^Yst5eIyW

The tangent and bitangent of each vertex is different
and thus we need to preform the transformation into
tangent space  ^DkBQYwty
